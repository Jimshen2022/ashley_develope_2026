Standard Operating Procedure

Replacement Parts (RP) Business Process

Version: 1.0 | Effective Date: April 2026

1. RP Receiving Process (Whse 335)

This section outlines the process for receiving replacement parts into Warehouse 335, utilizing AS.COM and HighJump (HJ) systems.

Step Activity Description System / Reference
1.1 SCP creates the RP order on AS.COM platform. Option 0683 / RPP070
1.2 Supplier processes the order and generates the Advanced Shipping Notice (ASN). ASN Generation
1.3 Container with RP orders arrives at the facility; perform ASN Check-In. AS.COM / Dock Check-in
1.4 System synchronization: Update PO on AshleySupplier and record RP transactions into Whse 335. AFIPROD (TRDATA)
1.5 HighJump (HJ) scans RP specific item numbers and quantities into HJ stock and Whse 335 default location. WMS Scanning
1.6 HJ moves the RP specific items from the default location to the dedicated storage location. RP998XL1

2. RP Pick & Pack Process

2.1 Order Entry & System Integration

Orders are entered via two primary channels and synchronized to AS400 core tables:

  • Ashley Direct: Orders entered via the web portal are written to SQL Server and submitted to AS400.
  • AFI AS400: Manual entry via Option 0065 (RPP015).
  • Data Storage: Core data is stored in ARPHEDR (Header), ARPDETL (Detail), ARPOCOM/ARPICOM (Comments), and VCBCTL.

Final stages involving trip consolidation and system-wide inventory/financial updates.
