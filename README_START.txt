🎯 工作原理
==================================================================================

1. 🔍 扫描源目录
   - 找到所有 .pbix 文件
   - 显示文件列表

2. 📁 创建目录结构
   - 为每个 .pbix 文件创建一个同名文件夹
   - 存放在目标目录下

3. 🔄 逐个转换
   - 打开 Power BI Desktop
   - 加载 .pbix 文件
   - 用户在 Power BI 中执行 "另存为" → 选择 .pbip 格式
   - 文件保存到对应文件夹
   - 脚本检查是否成功
   - 继续下一个文件

4. ✅ 完成
   - 显示转换统计
   - 提供后续指导


==================================================================================
📋 Power BI 中的操作步骤（重要！）
==================================================================================

当 Power BI Desktop 自动打开文件时，请执行：

步骤 1: 点击菜单栏中的 "文件"
步骤 2: 选择 "另存为"
步骤 3: 在"保存类型"下拉菜单中，选择：
        "Power BI 项目文件 (*.pbip)"
        ⚠️  注意：不要选 "Power BI 桌面 (*.pbix)"

步骤 4: 确认文件名和保存位置（脚本已指定）
步骤 5: 点击"保存"按钮
步骤 6: 等待文件保存完成（进度条消失）
步骤 7: 关闭 Power BI Desktop
步骤 8: 回到脚本窗口，按 Enter 继续

💡 提示: 每个文件会重复这个流程

==================================================================================
🐛 快速故障排除
==================================================================================

问题 1: Power BI 找不到
  → 检查开始菜单是否有 Power BI Desktop
  → 如未安装，从 https://powerbi.microsoft.com/downloads/ 下载

问题 2: 脚本执行权限错误
  → 右键 run_power_bi_converter.bat，选择"以管理员身份运行"
  → 或在 PowerShell 中运行 Set-ExecutionPolicy Bypass

问题 3: 文件未转换成功
  → 确保在 Power BI 中选择 .pbip 格式（不是 .pbix）
  → 检查保存位置是否正确
  → 查看日志文件了解详细错误信息

问题 4: 权限拒绝
  → 以管理员身份运行脚本
  → 检查目标目录权限

更多帮助: 查看 QUICK_START.md 或 POWER_BI_README.md
